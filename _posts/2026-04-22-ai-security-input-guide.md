ADS（AI駆動サービス）の課題を進めていると、避けて通れないのが**APIキーの取得と管理**です。
「どこに保存すればいいのか」「そもそも何に気をつければいいのか」が最初はよくわからず、少し調べ始めたのがこの記事のきっかけです。

加えて、フリーランスとして活動している立場から、もう一つ気になることがありました。
クライアントとのセッション内容には当然、守秘義務が生じます。一般的なプライバシーポリシーは用意しているものの、**明確なセキュリティポリシーがない**ことに気づいたのです。

「AIツールを使うなら、そのあたりをちゃんと整理しておかなければ」——そう思い立ち、リサーチした内容をまとめました。
個人事業主・フリーランスの方にも参考になれば幸いです。

---

## AIに入力してはいけない情報

### 1. 個人情報

本人特定につながる情報の組み合わせは特に危険です。

- マイナンバー・社会保障番号
- パスポート番号・免許証番号
- **生年月日＋氏名のセット**（単体より組み合わせのリスクが高い）
- 自宅住所・電話番号

> 「名前だけなら大丈夫」と思いがちですが、他の情報との組み合わせで本人特定リスクが一気に高まります。
{: .prompt-warning }

### 2. 認証・セキュリティ情報

漏れた瞬間に被害が始まる情報群です。

- パスワード・PINコード
- APIキー・シークレットトークン
- 二段階認証コード
- SSHキー・証明書の秘密鍵

> APIキーやトークンは、漏えい後に悪用されるまでの時間が非常に短いです。絶対に入力しないでください。
{: .prompt-danger }

### 3. 金融情報

- クレジットカード番号・CVV
- 銀行口座番号
- 暗号資産のシードフレーズ・秘密鍵

### 4. 機密ビジネス情報

- 未公開の財務データ・M&A情報
- 顧客リストや顧客の個人情報
- NDA（秘密保持契約）で保護された内容
- 社内の機密コード・知的財産

> クライアント情報をAIに貼り付ける場合は、NDAや契約内容との整合性を必ず確認しましょう。
{: .prompt-warning }

### 5. 医療・健康情報

- 診断結果・処方薬の詳細
- 他人の健康状態に関する情報

### 6. 他人のプライバシーに関わる情報

- 本人の同意なく共有される第三者の個人情報
- プライベートな会話のスクリーンショットなど

---

## なぜ避けるべきか

- 入力データがモデルの学習に使われる可能性がある（サービスによって異なる）
- データ漏洩・不正アクセスのリスク
- 個人情報保護法・GDPRなどの法令違反になる可能性

> **代替手段：** ダミーデータや匿名化した情報に置き換えてから質問すると、安全に活用できます。
{: .prompt-tip }

## 自動バリデーションの方法

> このセクションはリサーチ段階の備忘録です。実装方法はまだ検討中のため、「こういう手段がある」という情報整理として残しています。
{: .prompt-info }

うっかり入力を技術的に防ぐ手段は、用途・規模によって選択肢が変わります。

### 1. DLP（データ損失防止）ツール ── 企業向け

| ツール | 特徴 |
|---|---|
| **Microsoft Purview DLP** | Microsoft 365環境に統合、AIツールへの送信も監視可能 |
| **Google Cloud DLP** | テキスト中の個人情報を自動検出・マスキング |
| **Nightfall AI** | SaaS/API特化。Slack・ChatGPT等と連携可能 |

### 2. ブラウザ拡張機能 ── 個人向け・手軽

AIチャットの入力欄をスキャンし、クレジットカード番号やメールアドレス等のパターンを検出して警告を出してくれます。

代表例：**Prompt Security**、**Harmonic Security**

### 3. 正規表現ベースのフィルタ ── 自前で組む場合

よく使う検出パターンの例です。

```python
# クレジットカード番号
r'\b\d{4}[- ]?\d{4}[- ]?\d{4}[- ]?\d{4}\b'

# メールアドレス
r'[\w.-]+@[\w.-]+\.\w+'

# マイナンバー（12桁）
r'\b\d{4}\s?\d{4}\s?\d{4}\b'

# APIキー（一般的なパターン）
r'(sk-|api_key|token)[a-zA-Z0-9_-]{20,}'
```
{: .nolineno }

### 4. プロキシ／ゲートウェイ型 ── 企業ネットワーク全体を守る

| ツール | 特徴 |
|---|---|
| **Azure AI Content Safety** | API呼び出し前にフィルタリング |
| **Cloudflare Gateway** | AIサービスへの通信をDLPルールで検査 |
| 自社プロキシサーバー | AIへのリクエストを中間で検査・ブロック |

### 5. Git pre-commitフック ── 開発者向け

コードをAIに貼り付ける前に、ローカル環境でシークレットを検出します。

```bash
# gitleaksのインストール例（Homebrew）
brew install gitleaks

# truffleHogでリポジトリをスキャン
trufflehog git file://. --only-verified
```
{: .nolineno }

代表ツール：`gitleaks` / `truffleHog` / `detect-secrets`（Yelp製）

## どれを選ぶか

| 用途 | おすすめ |
|---|---|
| **個人・フリーランスで手軽に** | ブラウザ拡張機能 |
| **開発チーム** | pre-commitフック + 正規表現フィルタ |
| **企業全体** | DLPツール + ゲートウェイ型 |

> 完璧な検出は難しいため、**技術的な対策 ＋ 運用ルールの周知**を組み合わせるのが現実的です。
{: .prompt-info }

💡 **OHOポイント:** この記事は「技術メモ」であると同時に、「フリーランスのセキュリティ整備記録」でもあります。「プライバシーポリシーはあるけどセキュリティポリシーがない」という状況は、AIを活用し始めた個人事業主に意外と多いはず。まず「何を入力しないか」を決めることが、ポリシー整備の第一歩になります。
